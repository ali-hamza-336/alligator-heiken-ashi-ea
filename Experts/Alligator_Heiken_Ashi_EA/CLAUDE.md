## Cross-cutting invariants (enforce in every phase)

1. **Closed candles only.** Never `iClose(s, tf, 0)`; always shift `>= 1`. (spec Critical Reminder #2)
2. **No per-tick order modification.** SL/TP changes only inside the new-M15-bar handler in `OnTick`. (#7)
3. **Magic Number gate** ‚Äî every position lookup filters by `Magic_Number`. (#4, #6)
4. **Atomic state writes** ‚Äî write to `.tmp`, then `FileMove` to final. (#5)
5. **Risk progression is fixed array** `{0.30, 0.50, 0.70}`, never computed. (#1)
6. **Centralized time conversion** ‚Äî one `ServerToNY(datetime)` helper handling DST; no scattered offsets. (#3)
7. **Never move SL backward** ‚Äî every SL modify gated by `CTradeManager::IsImprovement`. (Phase 5)
8. **Server‚ÜíNY offset auto-derived per call** via `CurrentNYOffset()` ‚Üí `CSessionTime::DeriveOffsetHours`; `Server_To_NY_Offset_Hours` input is override-only (0 = auto). (Phase 6)
9. **Max 1 trade open at a time, globally** ‚Äî not per-symbol. `EvaluateOpenPosition` returns `true` (suppress all entries) whenever any position by `Magic_Number` is open; returns `false` only when `open_trade_ticket==0`. The per-symbol `hedge_clear` check is now redundant belt-and-braces. (Phase 6, spec ¬ß7)
10. **`CTradeManager::Decide` ignores the zero-close sentinel** ‚Äî `if(ctx.close_m15_s1 <= 0.0) return d;` after the time-based exits, before the price-based ones. Caller passes 0s for `close/lips/atr` when the bar event belongs to a different symbol than the open position. (Phase 6)
11. **Streak/cycle/daily-loss state mutates in exactly two places** ‚Äî `ResolveClosedPosition` (broker close: SL‚Üí`OnSLClose`, TP‚Üí`OnTPClose`) and `EvaluateOpenPosition` `MA_CLOSE_*` branch (`OnForcedClose` + `ApplyRealizedProfit`). Both followed by atomic `State.Save`. (Phase 6)
12. **Two new entry gates in `OnNewM15Bar`, both block-only (never force-close):** (a) total-DD emergency ‚Äî `CDailyLossManager::IsTotalDDBreached(equity, g_state.initial_balance, Max_Total_DD_Buffer)` right after the session gate, returns before any signal work; (b) per-symbol news ‚Äî `CNewsFilter::IsBlocked(canon, TimeCurrent(), News_Filter_Enabled, ‚Ä¶)` in the filter row, contributes `news_clear` to `all_pass`. News auto-skips in the Strategy Tester and is fail-open on calendar API errors. Both gates sit below `EvaluateOpenPosition`'s early return, so they only run when flat ‚Äî correct, since they gate *entries*. (Phase 7, spec ¬ß4.5 / ¬ß7)
13. **`EAState.initial_balance` is the FTMO max-loss baseline, snapshotted once and never recomputed** ‚Äî set in `OnInit` from `AccountInfoDouble(ACCOUNT_BALANCE)` only when ‚â§0 (fresh state / legacy file), then persisted. `IsTotalDDBreached` fails open when it's ‚â§0. Do not derive it from equity, high-water marks, or the `AccountSize_Reference` input. (Phase 7)
14. **Server‚ÜíNY offset is recomputed per call, not cached** ‚Äî `CurrentBrokerGMTOffsetHr(now)` does `MathRound((TimeTradeServer()‚àíTimeGMT())/3600)` each call (so a mid-run DST flip is picked up without restarting, and 12-month backtests stay correct across the whole period); in the Strategy Tester, if that comes out 0 (some MT5 builds make `TimeGMT()==TimeTradeServer()` there), it falls back to the EU-DST calendar via `CSessionTime::IsBrokerInDST` ‚Üí +2/+3. `CurrentNYOffset(now)` = `Server_To_NY_Offset_Hours` if non-zero, else `DeriveOffsetHours(now, CurrentBrokerGMTOffsetHr(now))`. The old `g_broker_gmt_offset_hr` global is gone. (Phase 8)
15. **Strategy Tester runs from fresh state and skips the heartbeat timer** ‚Äî `MQLInfoInteger(MQL_TESTER)` guards both: `State.InitDefault(g_state)` instead of `State.Load` (so a previous run's `EA_State_<magic>.json` in the agent dir doesn't leak streak/daily-loss/ticket across runs), and `EventSetTimer(900)` is only armed when *not* in the tester (OnDeinit flushes anyway). Live behaviour unchanged. (Phase 8)
16. **The Lips-break exit is softenable but defaults to the spec** ‚Äî `LipsBreak_ATR_Buffer=0`, `LipsBreak_Confirm_Bars=1`, `LipsBreak_Min_Hold_Bars=0` reproduces spec ¬ß3.4 exactly. `CTradeManager::Decide` applies them in order: min-hold gate (`bars_since_entry >= min_hold`) ‚Üí `IsBeyondLips(...,buf)` where `buf = mult√óatr_m15_s1` ‚Üí N-bar confirm (require s1 [+s2 [+s3]] all beyond). The zero-close sentinel guard (#10) stays above this block. EA fills the prior bars' Lips only when `confirm ‚â• 2/3`. Stage 2 Task C kept these gates but changed the **action** they trigger from `MA_CLOSE_LIPS` (market-close) to `MA_TIGHTEN_SL_LIPS` (SL tighten via `TightenLipsPrice`, clamped at entry, `IsImprovement`-gated). Post-Task-C, `LipsBreak_Confirm_Bars` default flips `1‚Üí2` in Task D ‚Äî bakes the Phase-8 tuned-good value into the code. (Phase 8 + Stage 2)

17. **Stage 2 `Decide` priority order** ‚Äî NYOpen ‚Üí Friday ‚Üí zero-close-guard ‚Üí `MA_PARTIAL_AND_BE` (only when `!ctx.partial_done` AND close reaches `entry ¬± trigger_R √ó entry_R_distance`) ‚Üí `MA_TIGHTEN_SL_LIPS` (only when `!ctx.partial_done` AND Lips-break confirmed AND `IsImprovement` on the tightened SL) ‚Üí `MA_TRAIL` (only when `ctx.partial_done` AND `bars_since_BE_move ‚â• trail_delay_bars` AND `IsImprovement`) ‚Üí `MA_NONE`. BE-done detection is `ctx.partial_done` (explicit flag), NOT `current_sl ‚â• entry` (the old derivation collapses post-partial when SL moves to ~entry). R for the +trigger_R compute is `ctx.entry_R_distance` (caller-snapshotted at entry time), NOT `MathAbs(entry - current_sl)` (which would also collapse post-partial AND post-tighten). (Stage 2 Task C)

18. **Partial-and-BE atomicity (Task D requirement)** ‚Äî when dispatching `MA_PARTIAL_AND_BE`, set `g_state.partial_done=true` + `g_state.be_move_time=bar_time` + `State.Save` **only after both `PositionClosePartial` AND `ModifySL` (with `tp=0` to clear the original TP) succeed**. If either fails, leave state untouched so the next bar can retry. Half-state (partial booked but SL not moved) would suppress further `MA_PARTIAL_AND_BE`/`MA_TIGHTEN_SL_LIPS` (both gated by `!partial_done`) AND require an artificially-elapsed `bars_since_BE_move` for the trail to fire ‚Äî runner ends up unmanaged. **Three sub-cases** by `PartialLot(...)` result: (a) returns 0 (Fraction=0 input): skip `PositionClosePartial` entirely, just `ModifySL(BE+buffer, tp=0)` + flag-set + State.Save (no streak hook, no profit booked yet); (b) returns `current_lot` (Fraction=1.0 OR close-full fallback): `PositionClose` (not partial) + `OnTPClose` + `ApplyRealizedProfit(pre_close_profit)` + clear `open_trade_ticket` + `partial_done=false` + `be_move_time=0` + State.Save (treats as a finished TP-win trade); (c) returns intermediate value: full sequence ‚Äî `PositionClosePartial` ‚Üí `ModifySL(BE+buffer, tp=0)` ‚Üí `OnTPClose` inline + `ApplyRealizedProfit(partial_profit)` ‚Üí `partial_done=true` + `be_move_time=bar_time` + State.Save (runner remains; trail handles the rest). (Stage 2 Task D)
